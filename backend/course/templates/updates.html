XSS vulnerabilities lead to account hijacks


Reference:https://portswigger.net/daily-swig/xss-vulnerabilities-in-google-cloud-google-play-could-lead-to-account-hijacks


A pair of XSS vulnerabilities were found in Google Cloud, Google Play, and DevSite, potentially allowing attackers to hijack accounts. The first was a reflected XSS bug on Google DevSite, where an attacker-controlled link could execute JavaScript on Google’s sites, bypassing security. The second was a DOM-based XSS vulnerability on Google Play, where attackers could manipulate the search page to execute malicious JavaScript and hijack accounts. The researcher, ‘NDevTK’, earned $3,133.70 for the DevSite bug and $5,000 for the Google Play vulnerability.

Command injection vulnerability in GitHub Pages


Reference:https://portswigger.net/daily-swig/command-injection-vulnerability-in-github-pages-nets-bug-hunter-4k


A security researcher, Joren Vrancken, discovered a vulnerability in GitHub Pages that allowed attackers to execute arbitrary code through the service’s build process. GitHub Pages, which uses Jekyll to build static websites, had an issue where an attacker could exploit the Theme Chooser functionality to run malicious code. By crafting a malicious URL and tricking an admin into interacting with it, the attacker could trigger a payload during the build process, potentially exposing private repositories. Vrancken reported the issue to GitHub, which promptly resolved it by removing the Theme Chooser feature. In recognition of his findings, Vrancken received a $4,000 bug bounty and a GitHub Pro subscription. He praised GitHub's bug bounty program, which he said combined traditional hacking techniques with GitHub-specific features. GitHub's security team commended Vrancken's work, highlighting the value of engaging with security researchers through the program.

Microsoft Edge deepens defenses against malicious websites


Reference:https://portswigger.net/daily-swig/microsoft-edge-deepens-defenses-against-malicious-websites-with-enhanced-security-mode


Microsoft has introduced an **Enhanced Security Mode** in Edge to protect users from potential threats when visiting unfamiliar websites. This feature disables certain security features, like JIT JavaScript compilation, and adds additional protections such as arbitrary code guards. It offers three settings: **Basic**, which adds security to less-visited sites; **Balanced**, which adapts based on user behavior; and **Strict**, which applies protections to all sites but may disrupt normal browsing. The feature is off by default, and users can add trusted sites to an exception list. It was introduced in Edge version 104. However, it doesn’t support WebAssembly (WASM) sites, which can be exempted.

Toyata seals Backdoor


Reference:https://portswigger.net/daily-swig/toyota-sealed-up-a-backdoor-to-its-global-supplier-management-network


A security researcher, Eaton Zveare, discovered a major vulnerability in Toyota's Global Supplier Preparation Information Management System (GSPIMS), which manages sensitive data for around 3,000 suppliers and 14,000 users. Zveare exploited a backdoor in the system's login mechanism to gain system administrator access. This allowed him to view and potentially modify supplier information, rankings, and sensitive employee comments, with the ability to delete, leak, or tamper with data. The vulnerability stemmed from issues in Toyota's Angular-based web application, allowing unauthorized access by manipulating JavaScript code and exploiting the "createJWT" API. Zveare reported the flaw to Toyota in November 2022, and the company promptly fixed it by blocking the exploit. While Toyota acted quickly, Zveare noted that a bounty reward would have been appreciated to further encourage security research.
